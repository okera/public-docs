Okera and Domino Data Labs Integration

Environment Setup

Domino has Environments, which are reusable environments that have any prerequisite software and settings installed on them. For example, the default “Domino Analytics Distribution” comes with Python 3.6 and R 3.4 installed, and can run Jupyter and Julia notebooks.

Typical usage of Okera inside Domino Data Labs is done using Python, leveraging the PyOkera library. To add this to an Environment, you simply to add the following to an Environment Dockerfile:

RUN pip install pyokera

To do this:

  1. Click on the Environments tab on the left sidebar.
  2. Select the “Domino Analytics Distribution py3.6 R3.4” default Environment.
  3. Click the “Duplicate Environment” option in the top right
  4. Rename from “Duplicate of Domino Analytics …” to “Okera-Enabled Domino Analytics …” by clicking on the name and editing the tex box.
  5. Scroll down to the Dockerfile definition and click “Edit Dockerfile”
  6. Leave the selection of “Custom Image” with the filled in text
  7. In the end of the Dockerfile instructions, add RUN pip install pyokera, then scroll to the bottom and click “Build”
  8. Once the image is built, in the Revisions page for your newly created Environment, click the Gear button next to the new revision and do “Set as Active”.

At this point, you have a new Environment that comes with PyOkera pre-installed. Note that this only has to be done once, typically by an administrator during initial set up.

Projects Setup

Projects are collections of individual workspaces and jobs, and are tied to a specific Environment. You can launch ad-hoc notebooks or run scheduled jobs in them.

You can create a new Project by:

  1. Click on the Projects tab on the left sidebar.
  2. Click on “New Project” in the top left, and give it a name (e.g. OkeraTest) and whether you want it public or private.
  3. Once it’s created, click on the Settings tab on the left sidebar and choose your previously created “Okera-Enabled” environment in the “Compute environment” setting. Make sure it is using the latest revision you built that includes PyOkera.

At this point, you’re ready to create a simple notebook that uses PyOkera. Note that many users can share a specific project and that this does not have to be repeated.

Workspace Setup

A workspace is just a running instance of a data science tool like Jupyter or RStudio. In this case, we will use Jupyter to show how to connect to your ODAS cluster.

To create a workspace:

  1. Click on the Workspaces tab on the left sidebar.
  2. Choose Jupyter and give your Workspace a name, like Okera Workspace. Note that it should already be pre-selected to use your Okera-Enabled Environment that you configured in the Project settings.
  3. Click “Launch Jupyter Workspace”, which will take you to the Jupyter File Viewer
  4. In the File Viewer, we will create a new notebook - click on “New” dropdown button near the top left and select “Python 3”.
  5. In the new Python Notebook, we will enter our script.

  6. First, enter:

     import okera.odas
 okera.odas.version()

    and press Shift+Enter. This should print out 1.4.0.

  7. Then, enter:

     from okera import context
 ctx = context()
 ctx.enable_token_auth(token_str='<your token>')

 with ctx.connect(host='<your planner host/ip>', port=12050) as conn:
     df = conn.scan_as_pandas('SELECT * FROM okera_sample.sample', max_records=10000)
     print(df)

    Make sure to replace <your token> and <your planner host/ip> with your JWT token information and your ODAS Planner IP. If the planner is running on a port other than 12050, replace that too. You can get these values from the ODAS UI (by clicking the About section at the top right) or by asking your administrator.

    Once you have the right values, press Shift+Enter. This should print out:

                                    record
 0      b'This is a sample test file.'
 1  b'It should consist of two lines.'

Note that you or your administrator can set the connection parameters (such as the Planner host and port) in the Project definition as Environment Variables so that they’re available to all workspaces and jobs.

Securing Your Token

It is not advised to put your token in every notebook you create, as it is both cumbersome and others may get access to it. Instead, you can configure it in your Domino account as a private Environment Variable that then becomes available in all notebooks and jobs that you run.

To set this up:

  1. Go to your Account Settings by clicking on the Account silhouette.
  2. Click on the “User Environment Variables” option or scroll down to it.
  3. Create a new Environment Variable called OKERA_TOKEN with the value being set to your token and click “Set Variable”.
  4. If you have a running Workspace, you’ll have to Stop it and then Relaunch it.

  5. Once it’s relaunched, navigate to the notebook you had previously created, and add this to the script:

     import os
 print(os.environ['OKERA_TOKEN'])

    and press Shift+Enter. This should print out your Okera token.

  6. Replace the line which had your token with:

     from okera import context
 ctx = context()
 ctx.enable_token_auth(token_str='<your token>')

    with

     import os
 from okera import context
 ctx = context()
 ctx.enable_token_auth(token_str=os.environ['OKERA_TOKEN'])

    and then re-run the cell using Shift+Enter. This should print out the same output as before, but it is now getting your token from the Environment Variable you set.
